# Day 23 - Light Mode and Dark Mode Implementation

**Date:** 22 July 2026

## Overview

Today was the twenty-third day of my Web Development Training. The focus of today's session was on enhancing the user experience of my Employee Management System by implementing **Light Mode** and **Dark Mode** themes.

To achieve this, I restructured my CSS by introducing **CSS Root Variables (`:root`)** for the default (Light Mode) color scheme and **Dark Mode variables** for the alternate theme. This approach made the styling more organized and reusable, allowing the entire application to switch between themes with minimal changes to the CSS.

Users can now easily toggle between Light Mode and Dark Mode, making the application more comfortable to use in different lighting conditions while giving it a modern and professional appearance.

## Workflow

### Theme Switching

1. User clicks the **Theme Toggle** button.
2. JavaScript changes the active theme.
3. The corresponding CSS variables are applied.
4. All pages instantly switch between Light Mode and Dark Mode.
5. The application maintains a consistent appearance throughout the selected theme.

## Key Learnings

- CSS variables make theme management simple and maintainable.
- The `:root` selector provides a centralized way to define reusable color values.
- Dark Mode can be implemented efficiently by changing variable values instead of rewriting CSS.

- Theme switching enhances the overall user experience and gives applications a modern look.
